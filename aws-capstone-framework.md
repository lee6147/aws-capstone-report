## 시스템 아키텍처

### 전체 흐름

```
사용자 → CloudFront+S3(React SPA) → API Gateway
  ├─ /api/upload → Lambda:upload → S3(PDF) → Bedrock KB → S3 Vectors
  ├─ /api/chat → Lambda:chat → Bedrock KB(RetrieveAndGenerate) → Claude Haiku
  └─ /api/documents → Lambda:documents → S3(PDF)
```

### 핵심 설계 결정 (ADR)

| 결정 | 선택 | 기각 | 근거 |
|:-----|:-----|:-----|:-----|
| 벡터DB | S3 Vectors | OpenSearch ($350/월) | 99.9% 비용 절감 |
| RAG | Bedrock KB | LangChain 직접 구현 | 코드량 80% 감소 |
| LLM | Claude 3 Haiku | Sonnet, Titan | 비용 최저 + 충분한 품질 |
| 임베딩 | Titan V2 | Cohere | AWS 네이티브, 최저가 |
| 프론트 | React + Vite | Next.js | SSR 불필요 |
| 백엔드 | Python 3.12 Lambda | TypeScript | boto3 기본 포함 |
| IaC | AWS SAM | CDK, Terraform | 학습 곡선 낮음 |

### AWS 서비스 구성

| # | 서비스 | 역할 | 과금 |
|:-:|:-------|:-----|:----|

## MVP 정의 (8주차)

### P0 필수 기능

1. PDF 업로드 (≤10MB)
2. 문서 인덱싱 (Bedrock KB 자동)
3. 질문-답변 (문서 기반)
4. 출처 표시 (페이지/문단)
5. 기본 채팅 UI

### 데모 시나리오

1. 브라우저에서 앱 접속 (CloudFront URL)
2. "생물학 교과서.pdf" 업로드
3. 인덱싱 대기 (진행 상태 표시)
4. 질문: "세포 분열의 단계를 설명해줘"
5. AI 답변 + 출처 (p.34, p.37 참조)
6. 추가 질문 → 답변 + 출처
